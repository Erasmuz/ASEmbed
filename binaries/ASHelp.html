ASEmbed Help Page¶

Configuring¶

Link software with mxmlc in 2 ways:
  • Add mxmlc to your system path
  • Click file->set mxmlc
    • Browse to and choose the folder containing mxmlc in the flex download.
      • Ex: Winodws: 
        C:\flexSDK\bin
      • Linux:
        /usr/flexSDK/bin

Support¶

Bitmaps¶

  • Supported Format Types:
    • PNG
    • GIF
    • JPG
    • JPEG
  • Example:
    package {

import flash.display.MovieClip;
import flash.display.Bitmap;
import testgraphics.TestGraphic;

public class Main extends MovieClip {
    public function Main() {
        //Create a new bitmap using the imported BitmapData and set is initial position
        var image:Bitmap = new Bitmap(new TestGraphic(40, 40));
        image.x = 400;
        image.y = 300;

        //Add the new Bitmap to the stage.
        this.addChild(image);
    }
}
}

Sprites¶

  • Supported Format Types:

    package {

import flash.display.MovieClip;
import testgraphics.TestGraphic;

public class Main extends MovieClip {
    public function Main() {
        //Create a new instance of the imported graphic and set initial position.
        var image:TestGraphic = new TestGraphic();
        image.x = 400;
        image.y = 300;

        //Add the new image instance to the stage.
        this.addChild(image);
    }
}
}

Naming Errors¶

Known Issues and Why Concerning Naming:¶

Files names to be embedded can not start with numbers
1 mxmlc assumes strings that start with digits are numbers, and causes it to crash.
File names to be embedded can not contain special symbols.
1 Only letters, digits, and underscores are allowed
2 mxmlc treats these as math symbols, or other special symbols and crashes.
All file names must be unique.
1 Unique in the entire build.
Files in sub-folders can't have the same name.
2 Not including the extension.
Ex: test.png and test.jpg fail.
3 ASEmbed uses the file name minus the extension as the class name to embed the object, This will cause an ambigious class name for mxmlc at compile time if two objects end up with the same name in the same library.
